QRM Business Inteligence

Cube

  • Cube is defined as set of dimensions and set of measures, which can be selected in BI Query
  • Measure - variable which is measurable, in general numerical value
  • Dimension - variable which can be used for grouping of records, for example reference to codetable, boolean value etc.
  • For each cube there will be a denormalized collection, which connects all data needed for query in flat form

User Possibilities

  • User can pick from several defined CUBES
  • When the CUBE is picked, he can pick dimensions for grouping and measures, which will be calculated for all combination of dimensions

Dimensions

  • can be picked two-dimensionally
    • row dimensions
      • column dimensions
  • dimensions can be hierarchical, they could be picked from tree
  • tree can have theese types of nodes:
    • group node - it is only container for dimensions (something like folder in folder structure), it cannot be dragged to table
    • hierarchic node - container which contains hierarchic dimensions sorted in hierarchic order from top to bottom, it can be dragged to table. When it does, the dimension defined on the top will be picked.
    • leaf node - does not contain anything else, can be dragged to table

Measures

  • can be calculated using several operators
    • sum - sum of measures grouped by all picked dimensions
    • count - count measures grouped by all picked dimensions
    • average - average of measures grouped by all picked dimensions
  • the operator has to be defined in stage of creating the cube

Filters

  • user can pick filter criteria to narrow the resulting grid
  • user can pick values of attribute, which are interesting for him (codetables or booleans)
  • user can enter numeric value range, which is interesting for him (numeric values, dates)
  • it is not neccessary to pick filters only from dimensions and measures. There could be wider or narrower set of filter criteria. Please pick from measures and dimension and add other neccessary attributes

Charts

  • selected data can be visualised using pie or column chart
