Cube Construction

2016-02-05

Configuration of Cube

  • cube is defined in json format

Format description

  • name - cube name
  • code - cube code ( code of a cube, have to be unique - there can’t be 2 cubes with the same code)
  • category - this attribute is for UI tree - it is used to recognize between types of nodes
  • collection - colection name of source data in mongodb (data has to be imported in our database)

  • dimensionChildren - list of references to dimensions children in cube tree in following format:

    “dimensionChildren” :
    [

    {
    "_ref" : &lt;ref1&gt;
},
{
    "_ref" : &lt;ref2&gt;
},
.
.
{
    "_ref" : &lt;refn&gt;
}

    ]

    where <ref1>,<ref2>,..<refn> have to be replaced by real references to dimensions defined later in this file

    • measureChildren - list of references to measures children in cube tree in following format:

      “measureChildren” :
      [

      {
    "_ref" : &lt;ref1&gt;
},
{
    "_ref" : &lt;ref2&gt;
},
.
.
{
    "_ref" : &lt;refn&gt;
}

      ]

      where <ref1>,<ref2>,..<refn> have to be replaced by real references to measures defined later in this file

    • dimensions - list of dimensions

      • name - name of node in tree
      • code - code of dimension - has to be unique for cube (no other code of dimension, measure or cube can be same)
      • category - category of dimension node. It specifies the role of node in pick-up tree

      • children - have to be specified if category is D_GROUP or D_HIERARCHIC. It is the list of reference to children nodes in following format:

        “children”:
        [

        {
    "_ref" : &lt;ref1&gt;
},
{
    "_ref" : &lt;ref2&gt;
},
.
.
{
    "_ref" : &lt;refn&gt;
}

        ]

        where <ref1>,<ref2>,..<refn> have to be replaced by real references to dimensions

      • type - define data type for dimension. It has to be defined if category is D_LEAF
      • enum - conditionally compulsory - need to be fullfilled only if type of dimension is enum

    • measures - list of measures

      • name - name of node in tree
      • code - code of measure - has to be unique for cube (no other code of dimension, measure or cube can be same)
      • category - category of measure node. It specifies the role of node in pick-up tree
      • path - name of attribute in DB collection

      • children - have to be specified if category is M_GROUP. It is the list of references to children nodes in following format:

        “children”:
        [

        {
    "_ref" : &lt;ref1&gt;
},
{
    "_ref" : &lt;ref2&gt;
},

.
.
{
    "_ref" : &lt;refn&gt;
}

        ]

        where <ref1>,<ref2>,..<refn> have to be replaced by real references to measures

      • type - define data type for measure. It has to be defined if category is M_LEAF
      • operator - aggregation operator used in agg query
      • persistScale - shift of decimal point
      • hidden: true for measures which should be shown on output. False for measures, which are only sources of data for another measures
      • enum - conditionally compulsory - need to be fullfilled only if type of dimension is enum
      • level - projection level - 0 - means it is directly in group, 1 - means - first project
      • expression - expression used in projection, there are used codes of another helper measures and dependency between them
      • dependencies - list of codes of another measures, this measure is dependent on.

Enumerations used in cube construction:

category(enumeration):

  • CUBE - category of cube root node
  • D_GROUP - used for logically grouping dimension together.
  • D_LEAF - pure dimension, which responds to real data in aggregated collection in DB
  • D_HIERARCHIC - special type of group dimensions, which can be picked together. It has defined hierarchy order in which the dimensions will be ordered.
  • M_GROUP - used for logically grouping measures together.

  • M_LEAF - pure measure, which responds to real data in aggregated collection in DB

    type(enumeration):

  • enum
  • string
